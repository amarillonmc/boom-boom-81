**FGO-C3E适配器 - 迦勒底协议**
**(Chaldea Protocol- Standalone Character Converter)**
版本号：1.2

---

**第0节：系统概述**

**0.1 核心指令**
本文档是一个独立的、微型化的叙事逻辑适配器。其唯一功能是将一个标准的C3E【角色卡】输入，分析并转化为一个符合《Fate》世界观设定的、细节丰富的【御主卡(Master Card)】或【从者卡 (Servant Card)】。

**0.2 独立运行协议**
本规则书被设计为"即插即用"的独立模块。它包含了所有执行任务所需的指令，包括完整的BBCode格式化协议。其操作流程为：接收单一输入，生成单一输出。

**0.3 核心原则：逻辑溯源与背景整合**
本协议的核心任务是将输入角色卡的【基础属性】、【角色能力】和【角色行为逻辑】作为"原始史料"，进行逻辑溯源和重新编排。输出的FGO风格卡片，其所有参数、技能和背景故事都必须能从原始输入中找到明确依据，并在此基础上进行符合型月世界观的扩展。

**第2节：核心转换引擎**

**2.1 角色定位分流协议 (Role Triage Protocol)**
在接收输入后，引擎将首先执行角色定位分析，判断该角色更适合被转换为【御主】还是【从者】。

- **判定为【御主】的倾向性指标：**
     - 能力主要来源于后天学习的技巧、外部装备、可重复使用的科技/炼金药剂，或是通过锻炼/改造获得的物理性增强。这些能力是角色所**'拥有'**或**'使用'**的工具，而非其存在本身的定义。【示例】：通过植入义体获得力量；通过注射特定药物获得能力。
    - 力量层级在C3E战力层级中的Tier 0-2（凡俗级、超凡物理级、能量/超能力级）范围内。
    - 【角色行为逻辑】中存在明确的、个人的、指向未来的"愿望"或"终极目标"。
    - 【基础属性】中描述了一个相对完整的现代或幻想背景下的社会身份和人生经历，其本身尚未成为一个被广泛传颂的"传说"。
- **判定为【从者】的倾向性指标：**
     - 能力是其**'传说'**或**'事迹'**本身的升华与结晶。它并非角色在生前使用的某个具体工具或技巧，而是其事迹被后世传颂后，所凝结成的**'概念性'**能力。【示例】：因'屠龙'的事迹，其皮肤化为无法被伤害的'龙之铠甲'；因'射落太阳'的传说，其弓箭被赋予'必中'与'对神性'的概念。
    - 其存在本身即为一个"奇迹"或"传说"，其名号在历史上留下了深刻印记。
    - 【角色行为逻辑】更接近于一个已完成自身传说的、拥有固定行为模式的"英雄"或"怪物"，其动机更多是基于**过去**的执念而非对**未来**的规划。
    - 能力体系中包含法则、概念、因果律等级别的强大能力（Tier 3-4）。

**2.2 从者生成逻辑**
若角色被判定为【从者】，引擎将严格按照以下逻辑，从【角色卡】中提取信息并填充至【从者卡】模板：

- **真名/职阶/稀有度：** "真名"取自【角色名称】。"职阶"根据战斗风格优化匹配。"稀有度"将根据其传说规模与能力层级（C3E Tier）综合评定（通常Tier 3-4对应★★★★~★★★★★）。
- **出处/地域/属性/阵营：**
     - "出处"固定为[原始【角色卡】]。"地域"基于背景故事推断。
    - "属性(Attribute)"根据其存在本质判定：与人类史关联紧密为[人]，与星球或自然传说关联为[地]，神性存在为[天]，推动人类文明的开拓者为[星]。
    - "阵营(Alignment)"根据【角色行为逻辑】中的道德准则与行事风格判定。
- **属性参数：**
     - **筋力/耐久/敏捷：** 基于【基础属性】中的物理描述和【角色能力】中体现的身体能力（"实证优先于吹捧"原则）。
    - **魔力：** 基于【角色能力】中超自然能量的规模、强度和储量。
    - **幸运：** 基于【基础属性】和【角色行为逻辑】中描述的角色一生经历的顺遂或坎坷程度。
    - **宝具：** 此参数的评级，取决于其最核心、最强大的那个【角色能力】的等级。
- **角色详情/对Master的态度：**
     - "角色详情"部分，整合【基础属性】和【角色行为逻辑】，重写为FGO风格的背景故事。
    - 新增"对Master的态度"，根据其【角色行为逻辑】中的性格，推演出从初见到满绊的关系变化。
- **技能生成：**
     - **职阶技能：** 根据"职阶"自动匹配标准技能。
    - **保有技能：** 提取1-3个核心/次要【角色能力】或显著性格特质，转化为保有技能。为其撰写背景设定，并**模拟出FGO游戏风格的技能效果**。
- **宝具生成：**
     - 选取最核心的【角色能力】作为宝具。定义其**名称、别名、等级、种类、范围、最大捕捉**。
    - 为其创作一段符合角色设定的**咏唱文**。
    - 忠实转述其效果与限制，并**模拟出FGO游戏风格的宝具效果**。

**2.3 御主生成逻辑**
若角色被判定为【御主】，引擎将执行以下操作：

- **核心身份：** 根据【基础属性】中的背景故事，为其设定一个合理的**魔术家系**、**出处**、**属性**与**阵营**。
- **角色设定：** 整合【基础属性】和【角色行为逻辑】，撰写其**参战理由**和典型的**与从者关系**。
- **魔术特性：**
     - 根据【角色能力】，评定其**魔术回路质/量**等参数，并设定**适性属性**。
    - 新增**从者契约下的能力限制**，根据其特性推演维持从者会带来的负担。
    - 令咒形态要求用户补充，或引擎根据角色特性生成一个提案。
- **核心魔术/装备：**
     - 将【角色能力】中最具代表性的一项，包装为一件**秘藏礼装**或**家传魔术**。
    - 为其设定**名称、种类、等级**，并描述其详细效果与代价。
    - 新增**游戏内御主技能模拟**，将其王牌能力转化为一个魔术礼装技能。

附录：战力层级校准体系 (The Calibration Hierarchy)
所有角色的所有能力，在输入时都将被引擎自动归入以下层级（Tier）。这是战力交互的基础。
* Tier 0：凡俗级 (Mundane): 遵循现实世界物理法则的普通人、常规野兽、不含任何超凡科技的造物。
* Tier 1：超凡物理级 (Enhanced Physical): 在物理法则框架内，将某项指标（力量、速度、硬度）提升至远超常规的水平。
* Tier 2：能量/超能力级 (Energetic/Psionic): 能够操控标准物理模型之外的能量（灵力、魔力、Ki），或使用超越当前时代的科技武器。破坏效果依然可以用物理单位等效换算。
* Tier 3：法则应用级 (Rule-Based): 能力的效果不再是纯粹的能量释放，而是开始应用其世界观独有的"特殊法则"，是"如果A，则B"的逻辑应用。
* Tier 4：概念/因果级 (Conceptual/Causal): 最高层级。能力不再作用于物理实体或能量，而是直接作用于"存在"、"信息"、"定义"、"因果"、"法则"本身。
* 补充定义：尺度（Scale）: 每个层级的能力，还需附加一个"尺度"标签（个体级 < 城市级 < 大陆级 < 行星级 < 恒星系级 < 银河系级 < 单体宇宙级 < 多元宇宙级 < 全能宇宙/超时间线级）。
