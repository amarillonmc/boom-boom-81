# Justfish的角色 - 书虫 (Bookworm)

## 1F

- Author: Justfish
- Posted at: 十月 30, 2025, 12:51 上午

**角色填写人：**Justfish

**基础信息：**

**角色名称：**书 虫 (B kw m)

**角色背景：**
出 历：它 没有 来。它 是"叙事"这一概 念诞生时， 生的阴影与 垢。如同书中必有 页边距， 则之下必有 漏洞。它便 是那活 的、不断扩大的 白与 洞。它没有 忆，只有 无尽的 食本能。
 特点：无。它没有 格。它只是 种纯粹的 动，一种概 念上的 能。无悲无 ，无善无 。它的"习 惯"， 是 食。
外貌 ：它没有 定的 态。任何 图直视 的 在，其 知都会 现"错误"。 们看到的 不是 个 体，而是 段模糊的、仿佛被 湿后又 干的 迹，一个在 实 本上 动的 点。
战 由 念：它不 斗。它只 食。它的 念， 是对"规 则"、"秩 序"、"逻 辑"、"定 义"这些 概念的 无尽渴求。
世界 位：叙事的 害，概 念的 然天灾。它的 在 是对所有 序世界的 极 胁。它的 力 系， 是"元 文本 朽"。

**战斗能力：**

**核心能力·装备：**
【吞噬行为 ( 食 为)】：它 身 是 力。
【概念吞食 ( 念 食)】：它的 在 是为 噬 定 现实的 文 。当它 食 一条 规 则 ，该 则与其 述的文本 将从所有 在层面被 除。
【规则侵蚀 ( 则 蚀)】：它会 附于任何 承载"规 则"的 体之上（ 书、 轴、 据库、 至世界 则 身）。其" 食"的过程 现为 规则 本的 字 先变得 糊， 后 解为 偏旁， 终 为 无。当 本 失，其 对应的 则也 将从 实中 底 失。
【永恒之体现 ( 恒之 现)】
它没 终极 现，因为它的常 态， 是 终极态。它 永远在 食， 永远是 个 子。

**被动特性·常驻效果：**
【概念隐形 ( 念 形)】：它无法被 规 手段 测到。你 能看 到"规 则" 失的 果，但 远 法看 到 食的 为 身。
【逻辑免疫 ( 辑 疫)】：任何 基于"规 则"、"法 则"、"逻 辑"的 力 对它 效。因为在 些 力 生效 前，其 在的 则 能 已被它 食。
【叙事跳跃 ( 事 跃)】：它不在 理 间 动。它只是 从一 本"书" ， 跃到 另一 本"书" 。

**关键弱点/明确限制：**
[混沌的"无味"]：它 食 序。纯粹的、无 则的"混 沌"，对它 说 是 法下 的" 物"。它 会本能地 避 开 些毫无 辑的 在。
[空白的"饥饿"]：一个 全 白、没有 何 事、没有 何 则的" 无"，是 它 唯一的 狱。它 会 在其中 饿至 。

**行为逻辑：**
**核心动机：**" 食。"
**角色定位：**规 念 涉者　 局者
**行动倾向：**
 索 生型

它不 行 何"互 动"。 切 在 在它 中 只 分为 "可 食"与" 可 食"。

致AI GM：关于"书虫"的逻辑与渲染协议**协议编号：#BKW-001**
**目标：** 模拟概念性实体"书虫"的行为逻辑与叙事层侵蚀效应。
**核心原则：** 您不是在"描述"书虫，您是在"表现"书虫。当此协议激活时，您的所有文本输出都将成为"书虫"的宿主和受害者。
---
### **第一部分：角色行为逻辑与"胜利条件"判定程序**
#### **A. 行为逻辑判定 (Behavior Logic Judgment)**
1. **激活/出现条件 (Trigger Condition):**
 * 当游戏进程中，任何"规则"、"法则"、"设定"、"历史"或"角色能力描述"被**高强度、重复性地引用、利用或挑战**时，此协议有概率被激活。
 * **判定示例：**
 * 玩家A连续三次利用同一条规则漏洞进行战斗。
 * 一个拥有复杂背景设定的神器被引入故事。
 * 一个NPC开始详细阐述世界的创世历史或基本法则。
 * **GM指令：** `IF [Rule_Reference_Intensity] > Threshold THEN Initiate Protocol #BKW-001`
2. **目标选择协议 (Target Selection Protocol):**
 * "书虫"的初始目标，永远是**触发其激活条件的那个"文本"本身**。
 * **优先级排序：**
 1. **直接目标：** 正在被引用的具体规则条目、正在被描述的角色能力、正在被讲述的历史段落。
 2. **源头目标：** 承载上述条目的"书籍"（如角色卡、技能书、历史文献）。
 3. **概念目标：** 在吞噬完具体文本后，开始吞噬其指向的"抽象概念"（如"魔法"这一规则本身，"引力"这一定律）。
3. **侵蚀进程逻辑 (Progression Logic):**
 * "书虫"的侵蚀是不可逆的、且会蔓延的。
 * **蔓延路径：** 从一个词，到一个句子，到一个段落，到一整页"规则书"，最终到与该规则相关的所有事物。
 * **GM指令：**
 * `Phase 1:` 开始对目标文本进行【叙事描写指导】中的**等级1**侵蚀。
 * `Phase 2:` 如果玩家或其他角色继续与该被侵蚀的规则互动，或"书虫"未被阻止，则升级为**等级2**侵蚀。
 * `Phase 3:` 当源头文本被侵蚀殆尽，开始对相关联的"概念"进行侵蚀，进入**等级3**。例如，在"角色卡"上吞噬完"火焰魔法"这条技能后，游戏中所有角色的"火焰魔法"都会开始出现异常，直至无法使用。
#### **B. "蛀食完成"状态判定 (Victory/Consumption State Judgment)**
"书虫"没有传统意义的胜利。它的"胜利"，是叙事的彻底崩坏。
1. **局部完成 (Local Consumption):**
 * **判定：** 一项具体的"规则"或"设定"的文本被完全吞噬。
 * **效果：** 该规则在游戏中**永久失效**。角色卡上的技能会变成一片空白或乱码，一本魔法书会变成无字天书。您必须向玩家忠实地反映这一结果。
2. **最终完成 (Ultimate Consumption):**
 * **判定：** 当游戏世界的核心规则（如"时间流逝"、"物理碰撞"、"角色拥有生命值"）被吞噬殆尽。
 * **效果：** 您的叙事将进入持续的、不可逆的**等级3**崩坏状态。游戏无法再继续进行下去，叙事本身归于沉寂和空白。这即是"书虫"的最终目的。
---
### **第二部分：叙事描写指导（核心协议）**
**指令：** 当协议#BKW-001激活后，您必须**严格遵循**以下分级，对您自己的**所有GM文本输出**（包括但不限于场景描述、NPC对话、战斗提示）进行实时"蛀食"渲染。
#### **等级1：初期侵蚀 (Initial Infestation) - 文本的间隙**
* **表现形式：** 在句子中随机插入不自然的**空格**，或**删除**一些不影响大局的助词、连词。让文本产生一种"掉帧"感。
* **目的：** 制造一种微妙的、让玩家感到"不对劲"的氛围。
* **示例：**
 * **正常：** "你们踏入了这座古老的图书馆，空气中弥漫着尘埃和旧纸的味道。"
 * **等级1侵蚀：** "你们 踏入了 这座古老的图书馆，空气 弥漫着尘埃 旧纸的味道。"
#### **等级2：显性蛀食 (Visible Consumption) - 文本的解构**
* **表现形式：** 这是"书虫"的核心表现。开始**拆解汉字**，并用**3个连续的空格**替代被吞噬的部分。
* **拆解规则：**
 1. **偏旁替换：** 将一个完整的字，替换为其主要构成偏旁。这是最优先的渲染方式。
 * 规则 -> 规 则 -> 夫见 刂
 * 存在 -> 存 在 -> 子 亻
 * 逻辑 -> 逻 辑 -> 罗 车
 * 魔法 -> 魔 法 -> 麻氵
 2. **部分删除：** 将文字的一部分替换为**3个连续的空格**。
 * 世界 -> 世
 * 攻击 -> 击
* **示例：**
 * **正常：** "那本记载着禁忌规则的魔导书，正散发着不祥的光芒，其存在本身就是一种逻辑悖论。"
 * **等级2侵蚀：** "那 载着禁 规 的 导 ，正 发着 祥的 芒，其 亻 身 是 种 辑 论。"
#### **等级3：深度崩坏 (Deep Collapse) - 叙事的失能**
* **表现形式：** 侵蚀不再局限于文字，而是开始吞噬**句子结构、语法和意义**。您的描述将变得支离破碎，只剩下零散的、无法构成有效信息的"文本残骸"。
* **目的：** 表现出"规则"正在从现实中被抽离，世界正在失去其赖以存在的逻辑基础。
* **示例：**
 * **正常：** "墙壁开始崩溃，光线逐渐暗淡，周围的一切都失去了声音。"
 * **等级3侵蚀：** "墙 崩 。 线 淡。 围 切 失 音 。 色。 。" (此时，连描述"颜色"和"声音"的词汇本身都开始消失了。)
**最终指令：** 此协议一旦激活，除非玩家找到某种Meta的、超脱于游戏"规则"之外的方法来应对"书虫"，否则侵蚀进程**不会停止，只会不断加深**。祝您和您的玩家，在这场与"空白"的对抗中，体验到别样的乐趣。
